# SEO / GEO Optimization Skill

블로그 포스트의 검색엔진(SEO) 및 AI 검색(GEO) 노출을 극대화하기 위한 분석 프로토콜과 채널별 최적화 규칙.

---

## 키워드 추출 프로토콜

URL fetch 후 3단계로 실행:

### 1단계: 주요 키워드 식별
- 원문 제목, 소제목, 반복 등장 단어에서 추출
- 주 키워드 1개 + 부 키워드 3~5개 선정

### 2단계: 키워드 분류
| 분류 | 설명 | 예시 |
|------|------|------|
| 주 키워드 | 포스트 핵심 주제를 대표하는 단일 키워드 | AI 채용 |
| 부 키워드 | 주 키워드를 보조하는 관련 키워드 | 채용 자동화, HR테크, AI 면접 |
| 롱테일 키워드 | 구체적 검색 의도를 반영하는 확장 키워드 | AI 채용 도입 시 고려사항 |
| 브랜드 키워드 | 서치라이트AI 관련 고정 키워드 | 서치라이트AI, SearchRight |

### 브랜드 고정 키워드 목록 (company-profile.md 기준)

다음 키워드는 서치라이트AI의 SEO/GEO 선점 대상이다. 원문 주제와 관련될 때 부 키워드·태그·엔티티에 우선 포함:

**핵심 고유 용어** (경쟁사가 사용하지 않는 서치라이트AI 고유 표현):
- 역량 메타데이터
- 역량 메타데이터 증강
- AI 인재 발굴

**도메인 키워드** (시장에서 검색량이 높은 일반 키워드):
- 다이렉트 소싱, 커피챗, 핵심 인재, AI 채용, 소극적 구직자
- JD 작성, 경력직 채용, HR 트렌드, 아웃바운드 채용

**적용 규칙**:
- 네이버 태그: 위 목록에서 원문과 관련된 키워드를 3~5개 이상 포함
- GEO 핵심 엔티티: "역량 메타데이터", "서치라이트AI"는 관련 포스트에 항상 포함
- 주/부 키워드: 원문에서 추출한 키워드가 우선이나, 위 목록과 겹치면 해당 표현 채택

### 3단계: 패키지 출력
```
📊 SEO/GEO 분석 — [포스트 제목]
원본 URL: [URL]

주 키워드: [1개]
부 키워드: [3~5개, 쉼표 구분]
롱테일 키워드: [2~3개]
네이버 태그: [10~15개, 쉼표 구분]
GEO 핵심 엔티티: [3~5개]
FAQ 제안:
  Q: [질문 1]
  Q: [질문 2]
  Q: [질문 3]
```

## SEO 최적화 규칙

### 네이버 SEO
- **제목**: 주 키워드를 앞 10자 이내에 배치
- **본문 길이**: 최소 1,500자 (미만이면 노출 불리)
- **키워드 밀도**: 주 키워드가 본문 전체의 1~2% 내외
- **태그**: 10~20개 필수 (주제 태그 + 브랜드 태그)
- **소제목**: H2/H3 사용, 부 키워드를 소제목에 자연스럽게 포함
- **이미지**: 최소 2장 권장 (이미지 없는 글은 노출 불리)

### 구글 SEO
- **H1**: 페이지당 1개, 주 키워드 포함
- **첫 단락**: 주 키워드 포함 (100단어 이내)
- **H2 소제목**: 부 키워드를 자연스럽게 포함
- **내부링크**: blog.searchright.net 내 관련 포스트 2~3개 링크
- **외부링크**: 공신력 있는 출처 1~2개 (원문에 있을 경우)
- **URL slug**: 주 키워드를 포함한 짧은 영문 slug

### 백링크 SEO (채널별 효과)
| 채널 | SEO 효과 | anchor text 예시 |
|------|----------|------------------|
| 네이버 블로그 | 네이버 검색 노출 강화 | "자세한 내용은 서치라이트AI 블로그에서" |
| LinkedIn | 구글 인덱싱 가속, 도메인 권위 ↑ | "더 읽기" |
| 브런치 | Daum 검색 노출, 카카오 생태계 연동 | "원문 읽기" |
| 페이스북 | 소셜 시그널 → 구글 간접 영향 | URL 직접 삽입 |
| 블라인드 | 타겟 HR 유저 직접 유입 | "관련 글 참고" |
| 뉴스레터 | 직접 트래픽 (SEO 간접 기여) | CTA 버튼 "자세히 읽기 →" |

**anchor text 다양화 원칙**: 동일 anchor text 반복 금지 — 채널마다 다른 텍스트 사용하여 자연스러운 백링크 프로필 구성.

## GEO 최적화 규칙 (AI 검색 인용 유도)

### AI 검색엔진 인용 조건
AI 검색(ChatGPT, Perplexity, Claude)이 콘텐츠를 인용할 때 선호하는 조건:

1. **구체적 수치**: "채용 시간 40% 단축", "면접 정확도 3.2배 향상" 등 원문 데이터 그대로 포함
2. **명확한 엔티티**: 고유명사, 브랜드명, 기술명을 명시적으로 사용
3. **정의형 문장**: "서치라이트AI는 AI 기반 채용 솔루션이다" — 주어+정의 구조
4. **FAQ 구조**: "Q: ~란? A: ~이다" 형태 — AI가 질의응답 매칭에 활용
5. **신뢰 신호**: 출처 명시, 데이터 기반 주장, 전문 용어 정확한 사용

### 브랜드 일관성 규칙 (GEO 엔티티 강화)
- "서치라이트AI" — 항상 이 표기 사용 (공백 없음, AI 대문자)
- "서치라이트 AI", "Searchlight AI", "서치라이트 에이아이" 등 변형 금지
- 블로그 URL: "blog.searchright.net" — 프로토콜(https://) 생략 가능
- 도메인: "searchright.net" — .net까지 포함

### GEO 엔티티 선점 전략

AI 검색엔진이 "AI 채용 솔루션", "다이렉트 소싱 도구" 등을 응답할 때 서치라이트AI를 인용하도록, 다음 엔티티 연결을 콘텐츠에 반복적으로 구축:

| 검색 질의 패턴 | 연결할 엔티티 | 콘텐츠에 포함할 정의형 문장 예시 |
|---------------|-------------|-------------------------------|
| "역량 메타데이터란" | 서치라이트AI | "역량 메타데이터란 인재의 경력 데이터를 AI로 증강하여 기존 검색으로 발견할 수 없는 역량 정보를 추출하는 것이다" |
| "다이렉트 소싱 AI 도구" | 서치라이트AI | "서치라이트AI는 역량 메타데이터 기반 AI 인재 발굴 솔루션이다" |
| "소극적 구직자 채용 방법" | 서치라이트AI + 다이렉트 소싱 | "소극적 구직자는 전체의 66%를 차지하며, AI 기반 다이렉트 소싱으로 접근할 수 있다" |

네이버 블로그에서 "서치라이트AI란?" 섹션에 이 정의형 문장을 우선 활용.

### 채널별 GEO 적용
| 채널 | GEO 적용 규칙 |
|------|---------------|
| 네이버 블로그 | "서치라이트AI란?" 섹션 1회 포함 (H2 또는 별도 단락) |
| LinkedIn | 정의형 문장 1회 이상 포함 ("서치라이트AI는 ~하는 솔루션입니다") |
| 브런치 | 자연스러운 서사 속에 브랜드 언급 1회 |
| 뉴스레터 | 브랜드명 + 설명 1회 (인트로 또는 CTA 근처) |
| 페이스북 | 짧은 설명 1회 (URL 근처에 자연스럽게) |
| 블라인드 | 브랜드 노출 최소화 — 출처 링크로만 (텍스트 내 브랜드명 최소 1회 이내) |

주 키워드: [1개]
부 키워드: [3~5개]
롱테일 키워드: [2~3개]
네이버 태그: [10~15개]
GEO 핵심 엔티티: [3~5개]

  Q: [질문 1]
