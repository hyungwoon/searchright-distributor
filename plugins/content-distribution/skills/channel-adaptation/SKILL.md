# Channel Adaptation Skill

원본 블로그 콘텐츠를 각 채널의 알고리즘·독자 기대·포맷에 맞게 재구성하는 베스트 프랙티스.

## 공통 규칙

- 원본 메시지(core claim) 변형 금지 — 톤과 구조만 채널에 맞게 조정
- 통계/데이터는 원문에서만 인용 (새 수치 창작 금지)
- 전 채널에 백링크(blog.searchright.net/...) 반드시 1회 이상 포함
- 브랜드명: **서치라이트AI** (공백 없음, AI 대문자)
- 각 채널 콘텐츠 독립 작성 — 다른 채널 문서를 참조하거나 교차 링크 금지

## 채널별 규칙 요약

| 채널 | 길이 | 톤 | 백링크 위치 |
|---|---|---|---|
| LinkedIn | 1,000~1,300자 | 전문적·인사이트형 | 마지막 줄 |
| 뉴스레터 | 400~600자(본문) | 간결·핵심만 | CTA 버튼 |
| 네이버 블로그 | 1,500~2,500자 | 정보형·SEO 최적화 | 도입부 또는 마무리 |
| 브런치 | 1,000~1,800자 | 에세이·서사형 | 마지막 "더 읽기" |
| 페이스북 | 200~400자 | 가볍고 공감형 | URL 직접 삽입 |
| 블라인드 | 600~1,000자 | 현직자 공감형 | "관련 글" 형태 |

## 1. LinkedIn

### 구조
```
[강렬한 첫 문장 — 통계/질문/대담한 주장]

[단락 1 — 문제·현상 제기]

[단락 2 — 핵심 인사이트·데이터]

[단락 3 — 결론·시사점]

👉 더 읽기: blog.searchright.net/[slug]

#해시태그1 #해시태그2 #해시태그3
```

### 규칙
- **길이**: 1,000~1,300자 (공백 포함)
- **첫 줄**: 스크롤을 멈출 수 있는 훅 (수치, 의외의 사실, 직접 질문)
- **본문**: 3~5단락, 줄바꿈(빈 줄)으로 가독성 확보
- **외부링크**: 본문 맨 마지막에 단 1개만 (알고리즘 페널티 방지 — 중간 삽입 절대 금지)
- **해시태그**: 3~5개, 맨 마지막 줄에 모아서 배치
- **금지**: "서치라이트AI가 ~" 식의 홍보 첫 문장, 링크 2개 이상

### 작성 팁
- "see more" 클릭을 유도하려면 첫 3줄이 핵심
- 개인 경험이나 관찰에서 출발하는 서두가 높은 반응
- 인사이트형 글이 노출 알고리즘에 유리 (단순 링크 공유 지양)

### 예시

**Good — 첫 문장**:
> "경력직 채용 공고를 올린 지 3개월. 지원자 47명 중 면접까지 간 사람은 2명이었습니다."

→ 구체적 수치 + 문제 상황 직면 → HR 리더가 "나도 이런 상황인데" 공감하며 멈춤

**Good — 전체 흐름**:
> "채용 시장이 바뀌고 있습니다. (훅)
>
> 지금 회사에서 찾는 인재의 66%는 채용 공고에 지원하지 않는 소극적 구직자입니다. (데이터)
>
> 저도 CPO를 채용할 때 같은 벽에 부딪혔습니다. 공고를 올리고 기다리는 전통적 방식으로는 시니어급 인재를 만날 수 없었죠. (경험)
>
> 결국 답은 다이렉트 소싱이었습니다. 핵심은 '검색되지 않는 인재'의 역량을 어떻게 파악하느냐입니다. (인사이트)
>
> 👉 더 읽기: blog.searchright.net/direct-sourcing-guide
>
> #다이렉트소싱 #AI채용 #HR"

**Bad — 첫 문장**:
> "서치라이트AI가 혁신적인 채용 솔루션을 출시했습니다!"

→ 홍보 어조로 시작 — 알고리즘 불이익 + 독자 이탈

**Bad — 전체 흐름**:
> "안녕하세요, 서치라이트AI입니다. 저희 서비스를 소개합니다. AI 기반으로 인재를 찾아드립니다. 무료 체험도 가능합니다! 지금 바로 사용해보세요."

→ 기업 계정 홍보 포스팅 — LinkedIn에서 가장 낮은 노출

## 2. 뉴스레터 (Stibee/Mailchimp)

### 구조
```
제목: [30자 이내 — 핵심 인사이트 or 숫자]
프리헤더: [50자 이내 — 제목 보완, 반복 금지]

[이름]님,

[인트로 1~2문장 — 왜 이 글이 중요한지]

• [핵심 포인트 1]
• [핵심 포인트 2]
• [핵심 포인트 3]

[자세히 읽기 → blog.searchright.net/slug]
```

### 규칙
- **제목(Subject)**: 30자 이내, 호기심 유발 또는 핵심 수치 포함
- **프리헤더(Preheader)**: 50자 이내, 제목과 다른 정보로 보완
- **본문**: 400~600자 (제목/프리헤더 제외)
- **CTA**: "자세히 읽기 →" 텍스트에 백링크 연결 (버튼 형태 권장)
- **개인화 변수**: `[이름]` 플레이스홀더 사용 → 발송 전 Stibee `${name}` 또는 Mailchimp `*|FNAME|*`로 대체 필요 (주석으로 안내)

### 작성 팁
- 제목에 숫자·구체적 수치가 포함되면 오픈율 상승
- 불릿 포인트 3개가 스캔하기 가장 좋은 구조
- CTA는 1개만 — 여러 개 넣으면 전환율 하락

### 예시

**Good**:
> 제목: 채용 공고 없이 시니어를 만나는 법
> 프리헤더: 소극적 구직자 66%에 접근하는 실전 전략
>
> [이름]님,
>
> 지금 올려둔 채용 공고에 딱 맞는 시니어가 지원할 확률, 생각보다 낮습니다.
>
> • 전체 구직자의 66%는 공고에 지원하지 않는 '소극적 구직자'
> • 다이렉트 소싱 시 개인화 메시지의 회신율은 평균 대비 3배
> • AI 기반 역량 분석으로 '검색 불가능한 인재' 발굴 가능
>
> 자세히 읽기 → blog.searchright.net/direct-sourcing-guide

→ 제목 23자, 호기심 유발. 본문 핵심 3개로 압축. CTA 1개.

**Bad**:
> 제목: 서치라이트AI 블로그 새 글이 업로드되었습니다! 꼭 읽어보세요~
> 프리헤더: 서치라이트AI의 새로운 소식을 전해드립니다

→ 제목 30자 초과, 브랜드명으로 시작, 내용 예측 불가 → 오픈율 하락

## 3. 네이버 블로그

### 구조
```
[SEO 제목 — 주 키워드 앞 10자 이내]

[이미지: 대표 이미지 설명]

## 소제목 1
[본문 200~300자]

## 소제목 2
[본문 200~300자]

### 하위 소제목
[본문]

## 소제목 3
[본문 200~300자]

## 서치라이트AI란?
서치라이트AI는 AI 기반 채용 솔루션으로, [원문에서 추출한 1~2문장 설명].
자세한 내용은 서치라이트AI 블로그(blog.searchright.net)에서 확인할 수 있습니다.

---
태그: #키워드1, #키워드2, ..., #서치라이트AI
```

### 규칙
- **길이**: 1,500~2,500자 (공백 포함)
- **SEO 제목**: 주 키워드를 앞 10자 이내에 배치
- **구조**: H2/H3 소제목 사용, 소제목당 200~300자
- **백링크**: 도입부 또는 마무리에 1회 ("자세한 내용은 서치라이트AI 블로그에서")
- **GEO 엔티티 강화**: "서치라이트AI란?" 또는 브랜드 소개 섹션 1회 필수 포함
- **태그**: 10~20개 (주제 태그 + 브랜드 태그)
- **이미지**: `[이미지: 설명]` 형태로 삽입 위치 표시 (최소 2장 권장)

### 작성 팁
- 네이버 검색 로직은 제목 앞부분 키워드에 높은 가중치
- 1,500자 미만은 네이버 노출에 불리
- 태그는 많을수록 노출 기회 증가 (20개 상한)

### 예시

**Good — 제목 + 도입부**:
> 제목: 다이렉트 소싱 실전 가이드 — 경력직 채용 성공하는 5단계
>
> [이미지: 채용 담당자가 노트북으로 후보자 프로필을 검토하는 장면]
>
> ## 다이렉트 소싱이란?
> 다이렉트 소싱은 채용 공고를 게시하고 지원을 기다리는 인바운드 방식과 달리, 기업이 직접 인재를 찾아 연락하는 아웃바운드 채용 방식입니다. 최근 HR 업계에서 다이렉트 소싱이 주목받는 이유는 명확합니다. 전체 구직자의 66%가 채용 공고에 지원하지 않는 소극적 구직자이기 때문입니다.

→ 주 키워드 "다이렉트 소싱"이 제목 첫 7자 이내. 정의형 도입부. H2에 부 키워드 포함.

**Bad — 제목**:
> 제목: 채용에 대한 이야기를 해볼까 합니다

→ 키워드 없음. 네이버 검색에서 노출 불가.

## 4. 브런치

### 구조
```
[제목 — 에세이 스타일, 호기심 유발]

[에피소드/질문/관찰로 시작하는 도입부]

[자연스러운 서사 전개 — 문제 인식 → 인사이트 → 생각의 확장]

[마무리 — 여운 남기기]

이 주제에 대해 더 깊이 파고든 글을 정리해뒀습니다.
→ [포스트 제목](blog.searchright.net/slug)
```

### 규칙
- **길이**: 1,000~1,800자
- **톤**: 에세이 스타일 — 개인 경험·관찰에서 출발하는 서사
- **구조**: 소제목 없어도 됨 (있다면 자연스러운 문장형, "H2 키워드" 스타일 X)
- **백링크**: 마지막 단락에 "더 읽기" 형태
- **금지**: 딱딱한 보고서 톤, 불릿 리스트 남발, "~입니다" 체 반복

### 작성 팁
- 브런치 독자는 "읽는 경험" 자체를 즐김 — 정보 전달보다 관점 공유가 핵심
- 에피소드(작은 이야기)로 시작하면 몰입률 상승
- 첫 문장에서 "이 글은 ~에 대한 글입니다" 식의 설명 금지

### 예시

**Good**:
> 어제 면접관으로 들어간 자리에서, 후보자가 물었다.
>
> "혹시 이 포지션, 어떤 문제를 풀려고 뽑는 건가요?"
>
> 순간 머릿속이 하얘졌다. 우리가 JD에 적은 건 "경력 5년 이상의 백엔드 개발자"였지, 그 사람이 풀어야 할 문제를 적진 않았으니까.
>
> 요즘 채용이 어려운 이유가 여기에 있다고 생각한다. 우리는 '스펙'으로 사람을 찾고, 인재는 '문제'로 회사를 고른다. 이 간극을 줄이는 것이 다이렉트 소싱의 본질이 아닐까.
>
> 이 주제에 대해 더 깊이 파고든 글을 정리해뒀습니다.
> → [채용 공고 작성 전, 반드시 체크해야 하는 것은?](blog.searchright.net/jd-checklist)

→ 에피소드로 시작, 서사적 흐름, 관점 제시, 자연스러운 백링크

**Bad**:
> 다이렉트 소싱의 정의와 장단점을 알아보겠습니다.
>
> 1. 다이렉트 소싱이란?
> 2. 장점 3가지
> 3. 단점 2가지
> 4. 결론
>
> 서치라이트AI는 이런 문제를 해결하는 솔루션입니다.

→ 보고서 톤, 불릿 리스트, 홍보 마무리 — 브런치 독자 기대와 완전히 불일치

## 5. 페이스북

### 구조
```
[질문형/공감형 첫 문장]

[핵심 인사이트 1~2줄]

blog.searchright.net/[slug]

[이미지 필수: 설명]

#해시태그1 #해시태그2
```

### 규칙
- **길이**: 200~400자
- **첫 문장**: 질문형("혹시 이런 경험 있으신가요?") 또는 공감형("채용이 이렇게 달라졌습니다.")
- **URL**: 본문 중간~하단에 직접 삽입 (링크 미리보기 자동 생성)
- **이미지**: `[이미지 필수: 설명]` 안내 포함 (이미지 없으면 노출 급감)
- **해시태그**: 1~3개만 (페이스북은 해시태그 과다 시 역효과)

### 작성 팁
- 페이스북은 짧은 글 + 이미지 조합이 최고 성과
- 외부 링크가 포함되면 알고리즘이 노출을 줄이므로, 본문 자체에 가치가 있어야 함
- 댓글 유도 질문이 인게이지먼트 상승

### 예시

**Good**:
> 채용 공고 올린 지 3개월째.
> 정작 만나고 싶은 사람은 지원하지 않는다는 걸 깨달았습니다.
>
> 실제로 구직자의 66%는 '좋은 기회가 오면 이직을 고려'하는 소극적 구직자.
> 이 사람들에게 먼저 다가가는 방법이 있습니다.
>
> blog.searchright.net/passive-candidates
>
> [이미지 필수: 소극적 구직자 비율 인포그래픽]
>
> #채용 #다이렉트소싱

→ 공감형 도입, 핵심 수치 1개, URL 자연 삽입, 이미지 안내 포함

**Bad**:
> 서치라이트AI에서 새로운 기능을 출시했습니다! AI로 인재를 찾아보세요. 지금 무료 체험 가능합니다! 🎉🎉🎉
> #서치라이트AI #AI채용 #HR #채용 #인재 #스타트업 #대기업 #무료체험

→ 광고 포스팅, 해시태그 과다(8개), 이모지 남발, 본문 가치 없음

## 6. 블라인드 / HR 커뮤니티

### 구조
```
[공감형 현상 제시 — 회사명 없이 시작]

[문제 분석 — 수치/데이터 활용]

[실용적 시사점 — 현직자가 바로 적용할 수 있는 관점]

관련해서 참고한 글 공유합니다.
blog.searchright.net/[slug]
```

### 규칙
- **길이**: 600~1,000자
- **카테고리**: HR/경력 카테고리 기준 (카테고리별 세부 규칙 상이함 — 작성 시 참고)
